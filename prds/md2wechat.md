# md2wechat — Markdown → 公众号排版工具 PRD

> Owner: Pichai
> 原始 PRD: Fabrice (md2wechat-prd.md)

## 概述

纯前端单 HTML 网页工具，从本地选择 Markdown 文件，实时预览并转换为微信公众号兼容的富文本排版，一键复制后可直接粘贴到公众号编辑器。

## 核心约束

- 不支持外部 CSS/JS，所有样式必须内联（inline style）
- 不支持 class/id 选择器
- 图片需 base64 或外链
- 代码块需特殊处理（公众号对 `<pre><code>` 支持有限）

## 功能清单

| # | 功能 | 说明 |
|---|------|------|
| 1 | 文件选择 | 点击按钮或拖拽上传本地 .md 文件 |
| 2 | Markdown 解析 | 支持标准 GFM（标题、列表、粗体、斜体、链接、图片、引用、表格、代码块、分割线、任务列表） |

| 4 | 内联样式渲染 | 所有 CSS 以 inline style 写入标签 |
| 5 | 一键复制 | 复制预览区的富文本 HTML |
| 6 | 代码高亮 | 代码块带语法高亮，内联样式着色 |

## 技术方案

- 纯前端，单 HTML 文件，无需后端
- Markdown 解析：marked.js（CDN）
- 代码高亮：highlight.js（CDN），渲染后 class 转 inline style
- 复制：Clipboard API，复制富文本 HTML
- 主题：CSS 变量 + JS 动态注入 inline style

## L1/L2/L3 验收描述

### L1 — 基础功能（7 条，必须全部通过）

- **L1.1 文件上传**：点击"上传文件"按钮 → 选择 .md 文件 → 左侧显示 Markdown 原文，右侧显示排版预览
- **L1.2 实时编辑**：在左侧编辑区手动修改文字 → 右侧预览在 1 秒内同步更新
- **L1.3a 标题层级**：上传包含 h1/h2/h3 的 .md → 三级标题字号递减、样式可区分
- **L1.3b 文本强调**：上传包含粗体和斜体的 .md → 粗体加粗渲染、斜体倾斜渲染
- **L1.3c 列表渲染**：上传包含无序列表和有序列表的 .md → 无序列表有符号标记、有序列表有数字编号
- **L1.4 代码块高亮**：上传包含 ```python 代码块的 .md → 代码块以暗色背景渲染，关键字有颜色区分
- **L1.5 一键复制**：点击"复制"按钮 → 粘贴到富文本编辑器 → 保留标题/正文/代码块的格式和颜色

### L2 — 交互体验 + 边界情况（7 条）

- **L2.1 拖拽上传**：将 .md 文件拖拽到页面 → 触发文件加载，效果与点击上传一致
- **L2.2 主题切换**：点击主题下拉 → 选择"优雅紫" → 预览区标题颜色、引用边框颜色立即变为紫色系
- **L2.3 字号调节**：将字号从 15px 调整为 18px → 预览区正文字号变大，布局无错位
- **L2.4 脚注转换**：上传包含 `[链接文字](https://example.com)` 的 .md → 正文中链接变为 [1] 上标，文末出现脚注列表
- **L2.5 表格渲染**：上传包含 GFM 表格的 .md → 表格有边框、表头加粗、隔行变色
- **L2.6 引用块样式**：上传包含 `> 引用文字` 的 .md → 灰底、左侧主色边框、字号略小
- **L2.7 自定义主色**：修改主色 → 标题、链接、引用边框等主色元素全部跟随变化

### L3 — 高级场景 + 鲁棒性（5 条）

- **L3.1 长文档性能**：上传 500+ 行的 .md → 3 秒内完成渲染，编辑时无明显卡顿
- **L3.2 特殊字符**：上传包含中文、emoji（😀）、HTML 实体（&amp;）的 .md → 全部正确渲染，无乱码
- **L3.3 空文件处理**：上传空 .md 文件 → 不报错，预览区为空白，功能按钮正常可用
- **L3.4 响应式布局**：将浏览器窗口缩小到 768px 宽 → 布局切换为上下分栏
- **L3.5 连续主题切换**：在 5 个主题间快速连续切换 → 预览区无渲染残留或样式错乱
