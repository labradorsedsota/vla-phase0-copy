# TripSplit — 旅行分账记账本 PRD

> Owner: Pichai
> Status: Draft

## 概述

多页面 Python Web 应用，用于旅行场景下的记账与分账。用户创建旅行、添加同行人、记录消费（含付款人与参与人），系统自动计算分摊金额并生成最优结算方案。

## 核心约束

- Python Flask + SQLite，单项目可独立运行
- 多页面路由（非 SPA）
- 不依赖外部 API 或第三方服务
- 无需登录/注册，本地使用
- 货币单位：人民币（¥），精确到分（两位小数）

## 页面结构

### Page 1: 旅行管理（首页）`/`

| 功能 | 说明 |
|------|------|
| 创建旅行 | 输入旅行名称、开始日期、结束日期、同行人列表（逗号分隔姓名） |
| 旅行列表 | 展示所有旅行卡片：名称、日期、人数、总支出 |
| 进入旅行 | 点击旅行卡片进入该旅行的记账页面 |

### Page 2: 记一笔 `/trip/<id>/add`

| 功能 | 说明 |
|------|------|
| 消费分类 | 下拉选择：交通、住宿、餐饮、门票/景点、购物、其他 |
| 金额 | 数字输入，精确到分 |
| 日期 | 日期选择器，默认当天 |
| 付款人 | 下拉选择（从同行人列表中选，即谁先垫付的） |
| 参与人 | 多选（哪些人一起消费的，默认全选） |
| 分摊方式 | 均摊（默认）/ 按比例 / 自定义金额 |
| 备注 | 可选文本输入 |

### Page 3: 账单明细 `/trip/<id>/list`

| 功能 | 说明 |
|------|------|
| 消费列表 | 按日期倒序展示所有消费记录 |
| 分类筛选 | 下拉筛选某一消费分类 |
| 编辑 | 点击编辑按钮修改已有记录 |
| 删除 | 点击删除按钮移除记录（需确认） |
| 汇总栏 | 顶部展示：总笔数、总金额 |

### Page 4: 分账结算 `/trip/<id>/settle`

| 功能 | 说明 |
|------|------|
| 个人汇总 | 每人：实际支付总额、应承担总额、差额（+应收 / -应付） |
| 结算方案 | 最少转账次数的结算指令列表，如「小明 → 小红 ¥128.50」 |
| 分类统计 | 各分类支出金额及占比 |

## 技术方案

- **后端：** Python Flask
- **数据库：** SQLite（单文件，随应用部署）
- **前端：** Jinja2 模板 + 原生 CSS（内嵌样式）
- **路由：** Flask 多路由多页面
- **分账算法：** 贪心法最少转账次数（净额排序 → 最大债权人与最大债务人配对消除）

## 数据模型

### Trip（旅行）
| 字段 | 类型 | 说明 |
|------|------|------|
| id | INTEGER PK | 自增主键 |
| name | TEXT | 旅行名称 |
| start_date | DATE | 开始日期 |
| end_date | DATE | 结束日期 |
| members | TEXT | 同行人列表（JSON 数组） |
| created_at | DATETIME | 创建时间 |

### Expense（消费记录）
| 字段 | 类型 | 说明 |
|------|------|------|
| id | INTEGER PK | 自增主键 |
| trip_id | INTEGER FK | 关联旅行 |
| category | TEXT | 消费分类 |
| amount | REAL | 金额（元） |
| date | DATE | 消费日期 |
| payer | TEXT | 付款人 |
| participants | TEXT | 参与人列表（JSON 数组） |
| split_mode | TEXT | 分摊方式：equal / ratio / custom |
| split_detail | TEXT | 分摊明细（JSON，ratio 或 custom 时使用） |
| note | TEXT | 备注 |
| created_at | DATETIME | 创建时间 |

## L1/L2/L3 验收描述

### L1 — 基础功能（7 条，必须全部通过）

- **L1.1 创建旅行**：在首页填写旅行名称「杭州三日游」、日期、同行人「小明,小红,小李」→ 点击创建 → 旅行列表中出现该旅行卡片，显示名称和 3 人
- **L1.2 添加消费**：进入旅行 → 添加一笔餐饮消费 ¥300，付款人小明，参与人全选 → 保存成功 → 账单列表出现该记录
- **L1.3 均摊计算正确**：添加 ¥300 消费，3 人均摊 → 分账结算页显示每人应承担 ¥100.00
- **L1.4 账单列表展示**：添加 3 笔消费后 → 账单明细页展示 3 条记录，总金额正确
- **L1.5 付款人差额计算**：小明垫付 ¥300（3 人均摊）→ 结算页显示小明应收 ¥200，小红应付 ¥100，小李应付 ¥100
- **L1.6 结算方案生成**：基于 L1.5 场景 → 结算方案显示「小红 → 小明 ¥100」「小李 → 小明 ¥100」
- **L1.7 多笔消费累计**：小明垫 ¥300 餐饮 + 小红垫 ¥600 住宿（均 3 人均摊）→ 结算页各人差额正确（小明: +100，小红: +400，小李: -200 → 待验算后修正）

### L2 — 交互体验 + 边界情况（7 条）

- **L2.1 按比例分摊**：添加 ¥400 消费，选按比例分摊，比例 2:1:1 → 结算显示承担金额分别为 ¥200、¥100、¥100
- **L2.2 自定义金额分摊**：添加 ¥500 消费，自定义分摊：小明 ¥250、小红 ¥150、小李 ¥100 → 结算金额与输入一致
- **L2.3 分类筛选**：添加餐饮和交通各 2 笔 → 筛选「餐饮」→ 只显示 2 条餐饮记录
- **L2.4 编辑消费**：修改一笔消费金额从 ¥200 改为 ¥300 → 保存后账单列表和结算金额同步更新
- **L2.5 删除消费**：删除一笔消费 → 确认后记录消失，总金额和结算方案重新计算
- **L2.6 部分参与人**：¥400 消费只选小明和小红参与（小李不参与）→ 均摊后小明和小红各承担 ¥200，小李承担 ¥0
- **L2.7 分类统计**：旅行有餐饮 ¥500、交通 ¥200、住宿 ¥800 → 分类统计显示各金额及占比（餐饮 33.3%、交通 13.3%、住宿 53.3%）

### L3 — 高级场景 + 鲁棒性（5 条）

- **L3.1 精度处理**：¥100 消费 3 人均摊 → 每人 ¥33.33，总和 ¥99.99，差额 ¥0.01 合理分配（不丢失不多出）
- **L3.2 多人交叉付款**：5 人旅行，多笔消费由不同人垫付、不同人参与 → 结算方案转账次数 ≤ 4（最少转账优化）
- **L3.3 空旅行处理**：创建旅行后不添加任何消费 → 账单页显示空状态提示，结算页显示全员 ¥0，不报错
- **L3.4 单人消费**：一笔消费只有 1 人参与且由该人付款 → 不影响分账结算（差额为 0）
- **L3.5 大量记录**：单次旅行添加 50+ 笔消费 → 列表页正常加载，结算金额正确
